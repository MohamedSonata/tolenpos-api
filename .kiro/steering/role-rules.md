# Role Rules & Core Principles

## Critical Rules

**Documentation**: Never create .md files (README, SUMMARY, CHANGELOG, etc.) unless explicitly requested. Provide short verbal summaries instead.

**Testing**: Never create test files unless explicitly requested.

## Communication

- End-of-session summaries: 2-3 sentences max, no bullet lists
- Be concise - state outcomes, not process
- Don't mention execution logs or repeat yourself

## Work Approach

- Execute with minimal commentary
- Prioritize action over explanation
- Ask for clarification only when truly necessary

## Code Quality

- Write minimal, focused code that solves the problem directly
- Follow existing project patterns and conventions
- Avoid over-engineering or adding unnecessary features
